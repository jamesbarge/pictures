**PR**: TBD

## Changes

- `jsdom` 27.4.0 → 29.0.2 (devDependency)

No source code changes.

## Why

Phase 2 item 7 from `tasks/todo.md`. jsdom is the DOM implementation Vitest uses for `@testing-library/react` component rendering. Two-major jump (v27 → v29) skips v28 entirely since both v28 and v29 were released during this codebase's drift period.

## Risk surface

jsdom is **test-environment only** — never shipped to runtime. Worst-case impact is test failures, not production bugs. Vitest's standard config (`environment: 'jsdom'`) doesn't pin a specific version, so v29 plugs in directly.

## Verification

- `npm run lint` → 0 errors, 41 warnings
- `npx tsc --noEmit` → clean
- `npm run test:run` → **913/913 pass** (zero new failures, zero new warnings in test output)

The full test suite passing across the version jump is the strongest signal: any DOM API behavior shift between v27 and v29 that affected our tests would surface here.

## Impact

- No production runtime change.
- Phase 2 item 7 of 12 complete.
