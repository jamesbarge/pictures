## How
JW3 uses **Spektrix** ticketing (client `jw3`) with a public read API at
`https://ticket.jw3.org.uk/jw3/api/v3`. Two calls, no browser:
1. `GET /events` — keep only `attribute_Genre == "Cinema"` (the programme also
   contains talks, languages, classes, music, walks — all excluded).
2. `GET /instances?startFrom=YYYY-MM-DD&startTo=YYYY-MM-DD` — join to the Cinema
   events by `event.id`; build one `RawScreening` per future, non-cancelled instance.

- **Dates**: `instance.startUtc` is UTC without a trailing `Z` — append `Z` before
  `new Date(...)`. No `ukLocalToUTC` needed (Spektrix pre-converts), so the BST
  off-by-one that PR #619 fixed for the HTML scrapers cannot occur here.
- **Booking**: `https://www.jw3.org.uk/spektrix/ChooseSeats?EventInstanceId=<id>` (verified HTTP 200).
- **sourceId**: `jw3-<instanceId>`. **Poster**: `event.imageUrl`. **Availability**: `instance.isOnSale`.
- Light `eventType` detection (q_and_a / intro / preview / premiere). NT Live and
  other live broadcasts carry `attribute_Genre = "Cinema"` and flow through; the
  data-quality pipeline classifies `content_type` downstream.
