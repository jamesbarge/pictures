## Context you must know before touching anything

1. **This repo scrapes ~26 London cinema sources into Supabase Postgres**
   (Drizzle ORM), matches films to TMDB, and enriches with Letterboxd
   ratings. Frontend is separate (`frontend/`, SvelteKit) — none of this
   work touches it.
2. **Scraping is local-Mac-only by owner decree.** Do not introduce Inngest,
   Trigger.dev, Vercel crons, or GitHub Actions schedules for scraping.
3. **Never commit to `main`.** Feature branches + conventional commits +
   squash-merge PRs. Update BOTH changelog locations on every PR
   (`RECENT_CHANGES.md` + `changelogs/YYYY-MM-DD-*.md`).
4. **Database rule**: scrapers add/update; never delete valid future
   screenings. The only sanctioned deletions are confirmed parse errors,
   true duplicates, orphans, and the guarded phantom-reconcile sweeps.
5. **Verification gates for backend changes**: `npm run test:run`,
   `npm run lint`, `npx tsc --noEmit` — all green before claiming done.
6. **The DB is fragile under concurrent write load.** On 2026-06-11 two
   consecutive `/scrape` runs wedged (50 min and 25 min of zero output,
   0% CPU, fresh connections fine — the run's own pool wedges on an await
   not covered by `withDbTimeout`). Plan 001 (circuit breaker, already
   written, TODO) addresses this. **Execute plan 001 before any plan here
   that runs the full pipeline.**

## What the audit found (one paragraph each)

- **TMDB matching is title-dominated and signal-starved.** The search is
  `searchFilms(title, hints?.year)` (`src/lib/tmdb/match.ts:165`). The
  director hint is threaded in but only used to pass an ambiguity gate —
  never in search or scoring. `runtime` and `original_language` are never
  examined (zero references in match.ts). Result: junk 2026 stubs beat
  1993 classics ("Cronos"), same-year same-title films are decided by
  popularity alone ("Dracula" → Besson instead of Radu Jude at the ICA),
  and docs collide with features ("Joyland" → a Kansas amusement-park doc,
  live in prod now).
- **The match audit trail is silently dropped.** `matchAndCreateFromTMDB`
  (`src/scrapers/utils/film-matching.ts`) computes `matchConfidence`,
  `matchStrategy`, `matchedAt`, `letterboxdUrl` and adds them to the
  in-memory cache record — but the `db.insert(films).values({...})` block
  above it **omits all four fields**. Only 43 of 685 matched upcoming films
  (4.3%) have any recorded confidence. This is a bug, not a design gap.
- **Year contamination**: some scrapers send the *screening* year as the
  film year. `createFilmWithoutTMDB` sanitizes this; the TMDB-matching path
  does not — so a 2026 screening year gives 2026 junk stubs a +0.3 exact-
  year bonus.
- **Letterboxd identity is guessed, and the one authoritative ID we touch
  is discarded.** Watchlist import extracts Letterboxd's canonical
  `data-film-slug` and throws it away; enrichment then guesses slugs from
  titles, **even for films with no TMDB match** — producing live wrong
  links ("Doctors Under Attack…" → `/film/gaza/`, "Nighthawks (1978)" →
  the 1981 Stallone film).
- **Scrapers starve the matcher.** Only 5–7 of 26 capture year/director;
  zero capture runtime — yet Rio, ICA, Curzon, and Garden all *parse*
  runtime today and drop it because `RawScreening` has no runtime field.
- **31% of upcoming films (306/993) have no TMDB match** and are never
  retried. Roughly half are recoverable (event prefixes, "(4K
  Restoration)" suffixes, plain titles like *Aliens* that would match
  trivially).
- **Phantom rows are structural.** 13/26 scrapers have no `sourceId`; the
  pipeline never deletes rows that vanish from the source. BFI Southbank
  has ~64 stale phantom rows + 33 near-duplicate pairs live now; a staged
  reconcile script exists at `src/scripts/_bfi_reconcile.ts`.
- **Operational debris from the 2026-06-11 runs**: enrichment phase never
  ran (both runs wedged before it); Close-Up + Cinema Museum unreachable
  3× ("Health check failed"); The David Lean Cinema is a true silent
  breaker (5 consecutive zero-yield runs, last good **never**); insert
  timeouts dropped ~19 screenings at electric-white-city and more at
  rich-mix/castle/the-nickel with no retry.
