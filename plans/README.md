## Findings considered and rejected

Recorded so they are not re-audited next run:

- **Festival-screening "race condition"** in `pipeline.ts` `linkScreeningToFestival`:
  the post-insert re-SELECT is keyed on the same `(filmId, cinemaId, datetime)`
  unique constraint the upsert targets — not a real race.
- **CSRF on user mutation routes**: Clerk session cookies are `SameSite` by
  default and sign-in is currently disabled site-wide; not worth a dedicated
  plan now.
- **"Add `limit=200` to the homepage screenings fetch"**: the homepage already
  trims to a 14-day window in `frontend/src/routes/+page.server.ts`; a blunt cap
  would silently drop screenings within that window.
- **"Dead `src/stores/`"** (as a standalone finding): the React stores ARE still
  imported by the legacy Next.js pages — folded into the larger legacy-UI finding above.
- **README `.env` instruction**: verified correct (`cp .env.local.example .env.local`).
- **localStorage JSON-validation hardening** in the Svelte stores: low value;
  fallbacks are already sensible.
- **Stray files** (`tsconfig 2.tsbuildinfo`, `.env.local 2.example`): deleted
  locally during the quick-win pass; they were untracked, not a repo problem.
